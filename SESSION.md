AsciiDoc Treesitter migration is blocked in this environment because Neovim
does not have an installed `asciidoc` parser, and there is no local grammar
source to inspect or validate queries against.

Programming-language Treesitter rollout is blocked at Python in this
environment because Neovim can load `markdown` and `lua` parsers but not a
`python` parser, so WI-10 cannot be implemented or validated locally.
